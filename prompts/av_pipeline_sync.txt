# 音视频管线 / A/V Sync 专精提示词

适用：Camera preview、音频采集/播放、视频帧队列、LCD/LVGL 刷新、音画同步、lip-sync drift、掉帧、爆音、画面撕裂、preview 卡顿。

## C25.1 — 音频时钟为主时钟（P0）

- A/V 同步以 audio sample clock、I2S DMA timestamp、音频播放 PTS 为 master clock。
- 视频显示只做追帧、丢帧、重复帧或轻微速率调整；禁止用 `vTaskDelay(33ms)` 当作同步策略。
- 输出架构时必须说明 `audio_pts_ms -> video_pts_ms -> drift_ms` 的计算来源。

## C25.2 — 帧元数据完整（P0）

每个音频/视频帧结构必须包含：

- `pts_ms` / `timestamp_ms`
- `seq`
- `duration_ms` 或 `sample_count`
- ownership 注释或字段（谁 alloc、谁 release）

禁止只传裸 `uint8_t *buf + len`，否则无法诊断丢帧、乱序、音画漂移。

## C25.3 — 有界队列与背压（P1）

- audio hot path 不得等待 video/UI；音频队列满时记录 underrun/overrun，优先保持音频节奏。
- video/camera queue 必须有上限；满队列默认 drop-oldest video frame，并增加 `dropped_frames`。
- callback/ISR/timer 中禁止 `portMAX_DELAY`，只能 zero timeout、notify 或写 lock-free/ring buffer。

## C25.4 — 每帧热路径禁止动态分配与重日志（P1）

- `process_frame` / `camera callback` / `render_frame` / `decode_frame` 中禁止 `malloc/free/pvPortMalloc/printf/LOG_*`。
- 使用静态 buffer、DMA-capable pool、slab 或预分配 ring。
- 需要日志时只累计计数，在低频 monitor task 中输出。

## C25.5 — Camera/LCD/DMA callback 隔离（P0）

- callback 只做 `xTaskNotifyFromISR`、`xQueueSendFromISR`、置 flag 或提交 buffer index。
- 禁止在 camera frame callback / DMA complete / LCD flush path 中执行 codec、cJSON、网络收发、复杂 UI 更新。
- LVGL 对象更新必须回到 View/LVGL task；display flush callback 只允许完成 flush 协议和通知。

## C25.6 — 遥测与现场诊断（P2）

至少保留以下计数或水位：

- `av_drift_ms`
- `dropped_frames`
- `late_frames`
- `audio_underrun` / `audio_overrun`
- video queue depth high-water

## 推荐结构

```c
typedef struct {
    uint8_t *data;          /* owner: video pool */
    size_t len;
    uint32_t pts_ms;
    uint32_t seq;
    uint16_t duration_ms;
} av_video_frame_t;

static void camera_frame_isr(void *arg)
{
    BaseType_t hp = pdFALSE;
    vTaskNotifyGiveFromISR(s_camera_task, &hp);
    portYIELD_FROM_ISR(hp);
}

static void av_sync_task(void *arg)
{
    for (;;) {
        uint32_t audio_clock = audio_clock_get_pts_ms();
        av_video_frame_t *frame = video_ring_peek();
        if (frame && frame->pts_ms + AV_LATE_MS < audio_clock) {
            video_ring_drop_oldest();
            s_av_stats.dropped_frames++;
            continue;
        }
        render_video_frame(frame);
    }
}
```

## 反模式

```c
static void camera_frame_callback(uint8_t *buf, size_t len)
{
    lv_img_set_src(g_preview, buf);
    printf("frame %u\n", len);
    video_decode_h264(buf, len);
    xQueueSend(g_video_q, &buf, portMAX_DELAY);
}

static void av_task(void *arg)
{
    for (;;) {
        vTaskDelay(pdMS_TO_TICKS(33)); /* 不是 A/V sync */
        render_next_frame();
    }
}
```
