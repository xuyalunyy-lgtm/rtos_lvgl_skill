# LVGL v8 vs v9 API 差异专精

## 警告

**v8 与 v9 不可混用 API。** 编译前确认 `lvgl.h` 中 `LVGL_VERSION_MAJOR`。
Agent 生成代码时必须先问用户 LVGL 版本，并只使用该版本 API。

---

## 版本识别

```c
#include "lvgl.h"
#if LVGL_VERSION_MAJOR >= 9
    /* v9 API */
#else
    /* v8 API */
#endif
```

---

## 核心架构差异

| 项目 | LVGL v8 | LVGL v9 |
|------|---------|---------|
| 显示注册 | `lv_disp_drv_t` + `lv_disp_drv_register()` | `lv_display_t` + `lv_display_create()` |
| 输入注册 | `lv_indev_drv_t` + `lv_indev_drv_register()` | `lv_indev_t` + `lv_indev_create()` |
| 刷新回调 | `disp_drv.flush_cb` | `lv_display_set_flush_cb()` |
| 双缓冲 | `lv_disp_draw_buf_init(&draw_buf, buf1, buf2, size)` | `lv_display_set_buffers()` |
| 主循环 | `lv_timer_handler()` | `lv_timer_handler()`（相同） |
| 异步 UI | `lv_async_call(cb, user_data)` | `lv_async_call(cb, user_data)`（相同） |
| 颜色 | `lv_color_hex(0xFF0000)` | `lv_color_hex(0xFF0000)` 或 `lv_palette_main()` |
| 样式 | `lv_obj_add_style(obj, &style, 0)` | 相同，部分 selector 变化 |
| 屏幕对象 | `lv_scr_act()` | `lv_screen_active()`（v9 推荐） |

## 显示驱动初始化

### v8

```c
static lv_disp_draw_buf_t draw_buf;
static lv_color_t buf1[LV_HOR_RES * 40];
static lv_color_t buf2[LV_HOR_RES * 40];

lv_disp_draw_buf_init(&draw_buf, buf1, buf2, LV_HOR_RES * 40);

static lv_disp_drv_t disp_drv;
lv_disp_drv_init(&disp_drv);
disp_drv.hor_res = LV_HOR_RES;
disp_drv.ver_res = LV_VER_RES;
disp_drv.flush_cb = my_flush_cb;
disp_drv.draw_buf = &draw_buf;
lv_disp_drv_register(&disp_drv);
```

### v9

```c
lv_display_t *disp = lv_display_create(LV_HOR_RES, LV_VER_RES);
lv_display_set_flush_cb(disp, my_flush_cb);

static lv_color_t buf1[LV_HOR_RES * 40];
static lv_color_t buf2[LV_HOR_RES * 40];
lv_display_set_buffers(disp, buf1, buf2, sizeof(buf1), LV_DISPLAY_RENDER_MODE_PARTIAL);
```

## 线程安全（v8/v9 通用）

- `lv_timer_handler()` 必须在专用 UI 任务
- 跨任务：`lv_async_call()` 或 mutex 包裹
- **禁止**在 WSS/音频回调直接 `lv_label_set_text`

见 [lvgl_thread_safety.txt](lvgl_thread_safety.txt)

## 常用控件 API 变化（v9 _rename 摘要）

| v8 | v9 |
|----|-----|
| `lv_scr_act()` | `lv_screen_active()` |
| `lv_disp_get_scr_act(NULL)` | `lv_screen_active()` |
| `lv_obj_clear_flag(obj, LV_OBJ_FLAG_xxx)` | 相同（部分 flag 名调整） |
| `lv_img_set_src(img, &img_dsc)` | 相同 |
| `lv_img_decoder` 自定义 | v9 图像解码架构重构，查阅 porting guide |

生成 v9 代码前，查阅官方 migration guide：
https://docs.lvgl.io/master/CHANGELOG.html

---

## lv_conf 关键差异

| 配置 | v8 | v9 |
|------|----|----|
| 内存 | `LV_MEM_SIZE` | `LV_MEM_SIZE` 或系统 malloc |
| 色深 | `LV_COLOR_DEPTH` | `LV_COLOR_DEPTH` |
| 刷新 | `LV_DISP_DEF_REFR_PERIOD` | `LV_DEF_REFR_PERIOD` |

---

## 平台 port 注意

| 平台 | v8 port | v9 port |
|------|---------|---------|
| ESP32 | `esp_lvgl_port` v8 分支 | 确认 IDF 组件版本 |
| STM32 | `lv_port_disp_template.c` | 需更新为 v9 display API |
| JL/BK | SDK 内置 LVGL 版本固定 | **不可擅自升级 major 版本** |

## Agent 检查清单

- [ ] 已确认 LVGL v8 或 v9？

- [ ] 未混用 `lv_disp_drv_t` 与 `lv_display_t`？

- [ ] lv_conf 与 SDK 自带版本一致？
