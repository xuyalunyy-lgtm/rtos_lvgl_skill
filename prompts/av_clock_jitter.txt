# 音视频时钟漂移 / Jitter Buffer 专精提示词

适用：长时间 lip-sync drift、网络音频抖动、A/V clock recovery、RTP/流媒体 jitter buffer、audio underrun/overrun、video late/drop/repeat、resample drift correction。

## C27.1 — 主时钟与 PTS 单调性（P0）
- 音视频同步必须选定唯一 master clock，默认用 audio sample clock / I2S DMA timestamp / audio PTS。
- audio/video frame 必须携带单调 `pts_us` 或 `timestamp_us`；禁止用 `xTaskGetTickCount()` 临时拼时间戳替代媒体时钟。
- tick count 只能做调度超时参考，不能作为媒体播放进度的真值来源。

## C27.2 — Jitter buffer 有界水位（P0）
- jitter/ring buffer 必须定义 capacity、low watermark、high watermark、target delay。
- 满水位策略必须明确：音频可丢旧包/补静音，视频可 drop-oldest；禁止无限扩容或 `portMAX_DELAY` 等包。
- 低水位时只做轻量补偿，禁止阻塞 audio hot path 等网络恢复。

## C27.3 — 漂移校正有上限（P1）
- drift correction 必须限幅，例如 `AV_DRIFT_PPM_LIMIT <= 1000`。
- 小漂移用 bounded resample ratio / video drop-repeat 修正；大漂移走 resync state，不在每帧 reset codec。
- 禁止把 `drift_ms` 直接乘成无界采样率调整或播放 delay。

## C27.4 — 迟到/过早帧策略（P1）
- video late frame：超过阈值 drop；小幅 early：repeat / hold last frame；audio underrun：insert silence。
- 禁止在 `render_frame` / `playback_frame` / `sync_frame` 中按 PTS 差值 `vTaskDelay(diff_ms)` 硬等。
- A/V sync task 可以有限 timeout 轮询，但 audio/camera/display per-frame 路径不等锁、不等网络。

## C27.5 — Underrun/Overrun 路径可预测（P1）
- underrun/overrun handler 只更新计数、插入静音、冻结/重复上一帧或触发低频 resync。
- 禁止在 underrun/overrun/jitter handler 中 `malloc/free/printf/LOG_*`。
- 恢复路径必须幂等，避免连续 underrun 时重复创建 buffer、codec 或 task。

## C27.6 — 漂移与水位遥测（P2）
至少保留：
- `drift_ms` / `drift_ppm`
- `jitter_depth` / `jitter_low_water` / `jitter_high_water`
- `underrun_count` / `overrun_count`
- `late_frame_count` / `dropped_frame_count` / `inserted_silence_count`
- `resync_count`

## 推荐结构

```c
#define AV_JITTER_CAPACITY_FRAMES 8U
#define AV_JITTER_LOW_WATERMARK   2U
#define AV_JITTER_HIGH_WATERMARK  6U
#define AV_TARGET_DELAY_MS        80U
#define AV_DRIFT_PPM_LIMIT        500

typedef struct {
    int64_t pts_us;
    uint32_t seq;
    uint32_t duration_us;
    uint32_t owner;
} av_video_frame_t;

static int32_t av_sync_clamp_ppm(int32_t ppm)
{
    if (ppm > AV_DRIFT_PPM_LIMIT) {
        return AV_DRIFT_PPM_LIMIT;
    }
    if (ppm < -AV_DRIFT_PPM_LIMIT) {
        return -AV_DRIFT_PPM_LIMIT;
    }
    return ppm;
}
```

## 反模式

```c
void av_render_frame(video_frame_t *frame)
{
    frame->pts_ms = xTaskGetTickCount();
    int drift = frame->pts_ms - audio_now_ms();
    vTaskDelay(pdMS_TO_TICKS(drift));
    printf("drift=%d\n", drift);
}
```
