# Kconfig / sdkconfig 密钥安全模板（C9）

## 原则

1. **入库 `config`**：只保留功能开关与非敏感默认值；`CLIENT_SECRET` 等留空 `""`
2. **本地 `config.secrets`**：仅含敏感 `CONFIG_*` 行，加入 `.gitignore`
3. **构建合并**：生成 `config.local`，设置 `CONFIG_SUBSTITUTE_FILE=config.local`（Armino/BK）
4. **示例文件**：`config.secrets.example` 可入库，值为空或占位符

## 目录布局（BK AVDK 工程）

```
projects/<app>/
└── ap/config/bk7258_ap/
    ├── config                 # 入库，SECRET=""
    ├── config.secrets.example # 入库模板
    ├── config.secrets         # 本地，gitignore
    └── config.local           # merge 生成，gitignore
```

## Kconfig 写法

```kconfig
config MY_API_CLIENT_SECRET
    string "API client_secret"
    depends on MY_FEATURE
    default ""
    help
        真实值写入 config.secrets，勿提交 config。
        见 config.secrets.example 与 scripts/merge_config_secrets.sh。
```

## 合并脚本（产品仓）

```bash
cp ap/config/bk7258_ap/config.secrets.example ap/config/bk7258_ap/config.secrets
# 编辑 config.secrets
./scripts/merge_config_secrets.sh ap/config/bk7258_ap
export CONFIG_SUBSTITUTE_FILE=config.local
make bk7258
```

## 审查命令

```bash
python skill/tools/secret_scan_checker.py --dir projects/
python skill/tools/secret_scan_checker.py --git-remotes
```

## 日志（C9.3）

- WiFi 密码：仅打印 SSID 或 `****`
- RTC/HTTP token：禁止 `LOGI("token=%s", ...)`
- 字典/开放平台 secret：禁止出现在串口与提交的 `日志.md`

## Git remote（C9.2）

```bash
# 错误
git remote set-url origin https://user:TOKEN@host/repo

# 正确
git remote set-url origin git@host:repo.git
# 或 HTTPS + credential helper（token 不入 URL）
```
