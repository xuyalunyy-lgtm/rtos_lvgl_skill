# 工作流：L3 LVGL 页面

将已确认的设计稿实现为目标固件工程中可维护的 LVGL 页面。本技能不提供内置代码生成器或模拟器。

## 输入

- 设计截图和目标显示尺寸
- LVGL 版本与目标工程构建命令
- 切图资源、字体、文本、页面状态和交互决策

## 交付内容

- 目标工程所需的页面 C/H 文件、资源和构建系统条目
- 显示假设、资源归属和交互行为的简明记录
- 来自目标工程的构建或硬件验证证据

## 实施流程

1. 从 [页面规划模板](../templates/lvgl_page_plan.json) 创建 `lvgl_page_plan.json`：声明显示尺寸/LVGL 版本/帧预算、唯一 UI owner、初始页，以及每页的状态、资源缓存策略、动态刷新区域和带 guard 的导航。
2. 运行 `python tools/lvgl_page_plan_checker.py lvgl_page_plan.json`；未通过时先修复规划，不进入页面编码。
3. 编码前确认坐标系、资源角色、字体、页面状态和运行时所有权。
4. 自适应分组优先使用 Flex 或 Grid。仅在有意还原设计稿时使用固定坐标，并在代码旁说明原因。
5. 在 GUI 上下文中调用 LVGL；后台任务更新必须通过工程既有的队列、Presenter 或异步机制投递。
6. 复杂美术使用资源文件；动态标签、控件和指示器使用 LVGL 控件。
7. 在目标工程中构建，并在其模拟器或设备上验证。浏览器 mockup 只能辅助布局，不构成视觉验证证据。
8. 记录目标设备上的帧率、`lv_timer_handler()` 最大耗时、flush 最大耗时和整屏失效刷新次数；低帧率时先缩小 dirty area、合并状态更新并检查 DMA/TE 同步，再调整刷新周期。

## 完成检查

- 所有资源和字体均能被目标构建解析。
- `lvgl_page_plan.json` 已通过校验，且所有页面、状态、资源策略和导航 guard 已在实现前确认。
- 页面能针对选定 LVGL 版本编译。
- 没有后台任务直接修改 LVGL 对象。
- 已验证显示尺寸、触摸行为和状态切换。
- 动态页面已在目标设备或模拟器上检查帧率与 flush 长尾；未测量时明确标记为未验证。
