# SDK 深度裁剪专精提示词

## 铁律

**在编写任何业务代码之前，必须先对原厂 SDK 做深度裁剪。**
但裁剪**没有固定模板** — 必须基于**用户产品的完整需求**量身定制。

Agent **必须先询问**用户完整产品需求，再制定裁剪方案；禁止套用通用删除列表。

---

## Step -1 — 询问完整产品需求（强制，先于一切）

用户未提供完整需求时，**必须**用结构化问题收集：

```markdown
### 产品需求问卷（Agent 必问）

1. **硬件平台与芯片型号**（如 AC792N / BK7258 / ESP32-S3）
2. **联网方式**（WiFi STA / BLE / 以太网 / 无网络）
3. **协议需求**（WSS / MQTT / HTTP / 自定义 UDP）
4. **音频**（Mic 采集 / 播放 / 编解码格式 OPUS/MP3/PCM / 回声消除）
5. **显示**（分辨率 / LVGL 版本 / 触摸 / 摄像头预览）
6. **存储**（SD / U盘 / Flash 资源 / OTA）
7. **蓝牙**（Classic / BLE / 无）
8. **RAM/Flash 预算**（如 512KB RAM / 4MB Flash）
9. **量产功能**（OTA / 产测 / 加密）
10. **低功耗**（用户自行设计 — Agent 仅 review，不预设方案）
```

**未得到明确答复的关键项，须标注假设并请用户确认，不得擅自删除可能需要的模块。**

---

## Step 0 — SDK 全景扫描（JL / BK 必须，其他平台强烈建议）

在动刀裁剪之前，**先整体扫描原厂 SDK**，建立模块地图：

### 扫描内容

```
1. 目录树：apps/ projects/ components/ middleware/ 职责
2. 任务表：task_info_table / xTaskCreate / 官方 Demo 启动了哪些任务
3. 配置入口：Kconfig / app_config.h / sdkconfig 全部 =y 项
4. 内存占用：编译一次，记录 Flash/RAM 基线（idf.py size / map 文件）
5. 依赖链：目标功能（如 WSS）依赖哪些子模块（LwIP → mbedTLS → wifi）
6. Demo 入口：main/app_main 初始化顺序
```

### JL / BK 扫描流程（强制）

```
Phase A — 只读扫描（不删代码）
  ├── 列出 apps/ 或 projects/ 下所有子工程
  ├── 列出 task_info_table 或等效任务注册
  ├── 导出 app_config.h / Kconfig 当前值
  └── 编译 Demo，记录 size 基线

Phase B — 需求映射
  ├── 用户需求 ↔ SDK 模块对照表
  └── 标记：必需 / 可选 / 可删

Phase C — 按需求裁剪（无固定顺序，依依赖链）
  ├── 先关叶子模块（Demo UI、CLI、未用 codec）
  ├── 再关独立协议（BT、MQTT...）
  └── 最后缩内存池和任务栈（细则 → memory_alloc_optimize.txt C7.2/C7.6）

Phase D — 每步验证
  └── 编译 + 最小闭环冒烟
```

扫描结果输出为 **SDK 模块地图**（Agent 必须生成）：

```markdown
| SDK 模块 | 路径 | 当前状态 | 用户需求 | 处置 |
|----------|------|----------|----------|------|
| audio_server | apps/audio | 启用 | 要 OPUS | 保留，关 MP3 |
| ble_task | apps/bt | 启用 | 不要 BT | 删除 |
| ... | ... | ... | ... | ... |
```

## 需求驱动的裁剪决策（非固定清单）

根据用户答复**动态生成**保留/删除表，示例：

**用户答：WiFi + WSS + OPUS Mic + LVGL 320x240，不要 BT/SD/OTA**

→ Agent 输出：

```
[保留] WiFi STA, LwIP, mbedTLS, WSS
[保留] audio_server enc OPUS, Mic ADC
[保留] LVGL v8, 320x240 单屏
[删除] 蓝牙 Classic/BLE 全栈
[删除] FAT / SD / U盘
[删除] OTA 分区（用户明确不要）
[删除] MP3/AAC/WMA 解码器
[删除] 官方 Demo 页 + CLI
[待确认] HTTP Server — 用户未提及，是否用于配网？
```

**禁止**在未询问 OTA/蓝牙/存储时默认删除或默认保留。

---

## 裁剪执行原则（顺序灵活，依依赖而定）

1. **先扫描，后动刀** — JL/BK 必须完成 Phase A 模块地图
2. **先配置后删源码** — Kconfig/宏关闭 → 编译通过 → 再物理删除
3. **依依赖链裁剪** — 删 WSS 则 mbedTLS 可能也可删；删 WiFi 则 WSS 必删
4. **每步一编译** — 禁止一次关十项
5. **实测后缩栈/堆** — `uxTaskGetStackHighWaterMark` / size 工具
6. **低功耗** — **用户自行设计**，Agent 不插入固定 sleep 策略，仅在用户提交方案时 review

## Agent 输出要求

SDK 搭建 / Demo 改造任务**必须**按序输出：

1. **产品需求问卷**（缺失项 + 假设）
2. **SDK 模块地图**（扫描结果）
3. **需求驱动的保留/删除表**（非模板）
4. **分步裁剪计划**（每步预期收益 + 风险）
5. **验证清单 + 测试模式宏建议**

**未完成 1 和 2，禁止写业务代码。**
