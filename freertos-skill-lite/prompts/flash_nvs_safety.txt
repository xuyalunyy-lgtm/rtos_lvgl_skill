# Flash / NVS / 状态持久化安全专精提示词

> 约束 ID：C19.1–C19.5 → [constraint_detail.md](../references/constraint_detail.md)

## 适用场景

NVS 读写、Flash 擦写、OTA 分区、掉电恢复、配置持久化、日志落盘。

---

## C19.1 — NVS 写入必须检查返回值（P0）

```c
/* ✅ 正确 */
esp_err_t err = nvs_set_str(h, "wifi_ssid", ssid);
if (err != ESP_OK) {
    LOG_E(TAG, "NVS write failed: %s", esp_err_to_name(err));
    nvs_close(h);
    return err;
}
nvs_commit(h);  /* 必须 commit 才落盘 */
nvs_close(h);
```

```c
/* ❌ 错误：不检查返回值，不 commit */
nvs_set_str(h, "wifi_ssid", ssid);
nvs_close(h);  /* 数据可能未持久化 */
```

---

## C19.2 — Flash 擦写期间禁止读取同分区（P1）

Flash 擦写（erase/write）期间，读取同分区会导致 undefined behavior 或 HardFault。

```c
/* ✅ 正确：擦写前关闭所有该分区的读句柄 */
nvs_close(read_handle);
nvs_flash_erase();  /* 擦写 */
nvs_flash_init();   /* 重新初始化 */
nvs_open("storage", NVS_READWRITE, &write_handle);
```

ESP32 的 `nvs_flash_erase()` 会清空**所有** NVS 数据，慎用。

---

## C19.3 — OTA 回滚验证（P1）

OTA 回滚必须在首次启动时确认固件有效，否则下次重启会回滚。

```c
/* ESP32 OTA 确认 */
#include "esp_ota_ops.h"

void app_main(void)
{
    esp_ota_mark_app_valid_cancel_rollback();
    /* ... */
}
```

未调用 `esp_ota_mark_app_valid_cancel_rollback()` 的固件，重启后会被回滚。

---

## C19.4 — 分区表一致性检查（P1）

OTA 产品必须有 `ota_0` + `ota_1` 分区；NVS 分区不可删除。

```csv
# partitions.csv 示例
nvs,      data, nvs,     0x9000,  0x6000,
otadata,  data, ota,     0xf000,  0x2000,
ota_0,    app,  ota_0,   0x10000, 0x1F0000,
ota_1,    app,  ota_1,   0x200000,0x1F0000,
```

```c
/* ❌ 错误：缺少 ota_1 分区，OTA 更新无法双分区切换 */
```

## C19.5 — Flash 写入寿命管理（P2）

Flash 擦写寿命有限（典型 10K–100K 次），高频写入场景须做磨损均衡。

```c
/* ✅ NVS 自带磨损均衡 */
/* 高频数据（如传感器日志）用 NVS blob API，避免频繁 set_str */

/* ❌ 错误：每秒写一次温度到 NVS */
while (1) {
    float temp = read_sensor();
    nvs_set_u32(h, "temp_x100", (uint32_t)(temp * 100));
    nvs_commit(h);  /* 每秒擦写一次 Flash */
    vTaskDelay(pdMS_TO_TICKS(1000));
}
```
