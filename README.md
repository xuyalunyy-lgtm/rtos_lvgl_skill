# FreeRTOS Embedded Architect Skill

Cursor Agent Skill: FreeRTOS IoT firmware architecture (MVP layering, LVGL thread safety, I2S DMA, A/V pipeline/A-V sync, codec format consistency, clock drift/jitter buffer, DMA cache/zero-copy buffer lifecycle, module contracts, task/queue topology, timeout budget, observability, lifecycle symmetry, hot-path exclusion, critical-path budget, copy budget, backpressure degradation, fault recovery, config matrix, one-click repro, regression samples, board resource contracts, lock budget and priority inversion prevention, critical section/interrupt mask budget, sensor integration contracts, OTA firmware update safety, cJSON leak prevention, WSS/mbedTLS, SDK trimming, error handling, state machines, logging standards, task priorities, timer management, multi-core IPC, low power, display drivers, hardware/software co-debug IO pin planning). Constraint system C1-C45 (currently 45 constraint domains, 248 rules).

## Four-Layer Structure

```
L0  SKILL.md                 Intent routing (<100 lines)
L0  agents/openai.yaml       Codex/OpenAI UI metadata
L1  workflows/               Orchestration (see workflows/README.md)
L2  references/              Overview / constraint matrix / structure
L3  prompts/ + platforms/   Scenarios + platforms (on-demand 1-3 + 1)
L4  examples/ + tools/       Full version exclusive
```

| Path | Responsibility |
|------|------|
| [workflows/](workflows/) | L2/L3/Debug/Self-iteration step orchestration |
| [agents/](agents/) | Codex/OpenAI UI metadata |
| [references/](references/) | core_rules, constraint_detail, skill_structure, release_governance, iteration_log |
| [prompts/](prompts/) | Scenario-specific chains (indexed by C domain, see skill_structure) |
| [platforms/](platforms/) | ESP32 / STM32 / JL / BK platform docs |
| [examples/](examples/) | Good/bad examples + `app_mvp.h` |
| [tools/](tools/) | Checker, codegen, fixtures, checker registry |
| [scripts/](scripts/) | Iterate, install, optional generated Lite sync (with `.cmd` wrappers) |
| Lite distribution | No longer maintained in source tree since v31; generated by release pipeline when lightweight package is needed |

Installation boundary: Cursor / Claude Code / Codex install scripts exclude repo-root `README.md`, `INSTALL.md`, `CHANGELOG.md`, `.github/`, `.vscode/`, `freertos-skill-lite/`, caches, and local SDK; the source repo retains these maintenance assets, and runtime indexes like `workflows/README.md` are preserved.
